Application Approval System

This was a college project. Responsible for the entire backend part, I created and managed REST APIs using NodeJS, ExpressJS & MongoDB. Front-End was developed by my friend using ReactJs. We also deployed this project to Render. Read More...

Designed for a college model, this project allowed a student to submit an application to his class coordinator (mentor) - by uploading PDF or Word files. The coordinator could approve, reject or forward the application to the next higher authority (Head of Department in this case). The student could track the status of his submitted application. This facility also existed for teachers and HoDs.

A system admin could add a principal, who would then add courses (e.g. B. Tech.), departments (e.g. CSE) along with HoDs. The HoDs could add classes along with coordinators (mentors) and teachers to their department. The mentor could add students to his class. Any higher-level authority could add an user (of lower-level) or entity based on permission. For example, the CSE HoD isn't allowed to add a teacher to IT Department, but the principal or admin is.

A higher-level authority could also send notices (circulars) to lower-level persons, based on permissions. For example, a HoD can send a notice to all the students in his department, but a mentor can send notice to his class only.
